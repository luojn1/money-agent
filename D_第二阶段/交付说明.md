## 6. 降级开关

| 开关 | 效果 |
| --- | --- |
| `DISABLE_CHAT=true`（后端） | `/chat` 返回 `mode:"disabled"` 的提示语，不含引用 |
| `VITE_DISABLE_CHAT=true`（前端） | 隐藏聊天入口，主报告不受影响 |
| 未配置 `LLM_API_KEY` | 走规则模板回答（默认演示态），仍带 citation |
| Mock 模式（`VITE_USE_MOCK_PIPELINE=true`） | 前端自动隐藏聊天入口（无真实报告可问） |

LLM 可选增强：配置 `LLM_API_KEY`（可选 `LLM_BASE_URL`、`LLM_MODEL`，OpenAI 兼容）后自动启用，
调用前对问题与上下文脱敏，15 秒超时或任何失败静默回退模板文案——与 D 的 `engine/llm_polish.py` 同套约定。
LLM 模式支持多轮追问：每次调用携带最近 8 条对话历史（脱敏后），用户接着上一轮补充信息
（如先问"还款压力大吗"再补"我工资15000"）时能延续话题结合报告数字估算；跑题问题礼貌拉回合同。
同一个 key 也会启用报告"建议落地"文案润色（D 的 `engine/llm_polish.py`，整合管线已传 `--llm`）：
结构、优先级、风险关联仍由规则引擎决定，LLM 只改写表达，实测 D 阶段含润色约 7 秒。

### LLM 密钥怎么配（给 A / 部署负责人）

密钥是**后端环境变量**，访问网页的用户（含老师）无需任何配置，打开即用。
当前已申请 DeepSeek Key 并先充值 10 元测试，LLM 模式已实测跑通（2026-07-10，单次问答约 2 秒，
余额用尽会自动回退模板回答，不影响演示）：

- **本地演示**：仓库根有 `start-demo.sh` 一键脚本；把密钥写进 `.env.local`（已在 .gitignore，
  不会提交）后执行 `./start-demo.sh` 即可，未配置密钥时自动用模板回答。
- **线上（腾讯云 CloudBase 云托管）**：控制台 → 对应服务 → 环境变量，新增
  `LLM_API_KEY` / `LLM_BASE_URL=https://api.deepseek.com/v1` / `LLM_MODEL=deepseek-chat`，
  重新部署后所有访问者的对话即为 LLM 增强。密钥不要写进 Dockerfile 或提交到仓库。

## 7. 已知问题 / 边界

- 检索为规则实现（类别关键词 + 2-gram 重叠 + 风险等级兜底），对高度口语化或跨风险的复杂问题可能只命中主风险；
  可后续接 C 的 RAG 检索增强，接口不变。
- 对话历史存后端内存（与 `pipelineTaskStore` 一致），服务重启即清空，演示足够，未做持久化。
- 姓名脱敏只覆盖"姓名/借款人/甲乙方/客户 + 2~4 字中文"这类显式标注，裸姓名不做启发式误伤；手机号/身份证/银行卡为强规则遮蔽。
- 2026-07-10 补充：优化了 D 自己的 `engine/recommender.py` 建议文案（动作化、按类别差异化理由、
  同类建议合并，与问题清单零重复），协议字段与 ID 关联规则不变，pytest 16 项与 verify:bcd-pipeline 均通过。
  其 LLM 层（`llm_polish.py`）不接触用户 PII。
