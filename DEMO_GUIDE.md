# 本地 Demo 演示指南

这份 Demo 用来让你亲自粘贴或上传合同，并看到 B → C → D 的完整分析结果。

## 1. 启动方式

推荐直接双击：

```text
demo/start_demo_server.bat
```

启动后不要关闭弹出的命令行窗口，然后在浏览器访问：

```text
http://127.0.0.1:8091/demo/index.html
```

也可以在项目根目录手动执行：

```powershell
C:\Users\asus\anaconda3\python.exe demo\demo_server.py --host 127.0.0.1 --port 8091
```

## 2. 页面怎么用

1. 点击左侧的“普通消费贷 / 信用卡分期 / 教育培训贷”任一测试合同。
2. 或者把自己的合同文本粘贴到文本框。
3. 也可以上传 `.txt` 文件。
4. `.pdf` 文件仅支持可复制文字的 PDF；扫描件需要先 OCR。
5. 点击“开始分析”。
6. 右侧会显示 B 解析中 → C 风险识别中 → D 建议生成中。
7. 完成后默认看到的是可视化报告，不会直接展示 JSON。
8. 如果需要排查技术链路，可以展开底部“技术详情”查看 B/C/D JSON 和 C Trace。

## 3. 测试合同 1：普通消费贷

```text
个人消费借款合同

甲方因日常消费向乙方申请借款本金10000元，借款期限12期，名义年利率12%。乙方同意放款，但需在放款时一次性扣除服务费500元、账户管理费300元，实际到账金额为9200元。甲方仍应按本金10000元计算还款义务，每月还款980元。

甲方提前还款的，应提前7日申请，并支付提前结清手续费200元，已收取的服务费、管理费不予退还。逾期还款的，乙方有权按正常利率1.5倍计收罚息，并收取违约金。甲方授权乙方及合作支付机构从绑定银行卡自动扣款，并同意乙方将逾期信息报送征信机构。
```

预期结果：

- 会识别为个人消费贷。
- 重点风险包括服务费/管理费、实际到账低于本金、真实年化偏高、提前结清手续费、逾期罚息、自动扣款授权。
- 摘要结论大概率为“先别签”或“谨慎签”。

## 4. 测试合同 2：信用卡分期

```text
信用卡账单分期业务协议

持卡人申请将本期信用卡账单12000元办理12期分期。本业务宣传为“0利息、免息分期”，但持卡人须按每期72元支付分期手续费，手续费按期计收并计入账单。分期交易将占用信用卡可用额度，额度随每期还款逐步恢复。

持卡人如提前还款或提前结清，需一次性偿还剩余本金，并支付剩余全部分期手续费，已收手续费不退还。若持卡人只偿还最低还款额，未全额还款部分将自交易日起按日计收循环利息。银行可通过短信、App或账单通知调整展示方式。
```

预期结果：

- 会识别为信用卡分期。
- 重点风险包括“免息但收手续费”、提前还款仍收剩余手续费、最低还款循环利息、额度占用。
- D 建议会提示把免息换算为真实总费用，并确认提前还款是否退手续费。

## 5. 测试合同 3：教育培训贷

```text
教育培训服务及学费分期协议

学员报名参加“职业技能提升课程”，总学费19800元，由合作金融机构提供学费分期贷款，贷款一次性划付至培训机构，学员按12期每期1830元还款。培训机构宣传“先学后付、包就业、不过包退”，但就业结果以市场情况为准。

学员退课、缺课或对课程质量不满意的，已开课后概不退款；如确需退费，应先扣除课程服务费、资料费和违约金。培训服务合同解除、课程暂停或培训机构停止经营的，不影响贷款合同继续履行，学员仍需按期还款。逾期将影响征信，并可能联系紧急联系人。
```

预期结果：

- 会识别为教育培训贷。
- 重点风险包括培训服务与贷款绑定、退费条件苛刻、服务停止后贷款仍需偿还、诱导性营销、逾期影响征信。
- D 建议会提示先确认退课、停课、机构跑路时贷款如何处理。

## 6. 结果怎么看

### 摘要

先看顶部一句话结论，例如：

```text
先别签：一年实际成本较高，有多项高风险
```

这里适合 30 秒快速判断合同是否值得继续谈。

### 核心数字

- 真实年化：越高越要小心。
- 风险分：分越高越需要谨慎。
- 额外支付：除本金外你可能多付的钱。

### 下一步行动

这里是 D Agent 生成的建议。演示时可以直接拿这些问题去问机构：

- 费用是否全部列明？
- 提前还款是否退费？
- 退课后贷款是否还要还？
- 逾期是否影响征信？

### 可视化报告

页面默认展示普通用户能看懂的报告：

- 总体结论：用红/黄/绿状态点标出是否建议签约。
- 核心数字：实际到账、总还款、额外支付成本、真实年化。
- 风险清单：按高风险优先排序，点击可展开合同原文、法规和案例。
- 下一步行动：可以直接复制给机构或朋友的确认问题。
- 合同原文定位：点击“查看合同原文”会高亮风险藏在哪句话里。
- 数据来源：底部标注用了多少条规则、法规和案例。

### 技术详情

底部折叠区用于核对技术链路：

- B 输出：合同解析、成本测算、条款数组。
- C 输出：风险项、风险统计、证据条款。
- C Trace：命中的规则、检索到的法规和案例、知识库使用情况。
- D 输出：建议和问题清单。

## 7. 输出文件位置

每次分析都会生成一个目录：

```text
demo/outputs/demo_时间戳_xxxxxx/
```

里面包括：

- `contract.txt`
- `b-output.json`
- `c-output.json`
- `c-trace.json`
- `d-output.json`
- `summary.json`

这些文件可以发给 A/B/C/D 同学排查链路。
