# 合同知识库（B同学：合同解析 Agent）

## 1. 目标

把消费贷/分期合同从自然语言文本转换为结构化字段，为成本测算 Agent、风险识别 Agent 和建议生成 Agent 提供可靠输入。

核心原则：只提取合同中出现或可由合同明确计算的信息；缺失字段必须标记为“未识别/合同未约定”，不能由模型猜测。

## 3. 标准抽取字段

### 3.1 基本主体

| 标准字段 | 类型 | 常见表述 | 说明 |
|---|---|---|---|
| institution | string | 贷款人、乙方、银行、消费金融公司 | 放款/授信机构 |
| borrower | string | 借款人、甲方、客户、持卡人 | 可脱敏保存 |
| contractType | enum | 个人信用消费贷款、个人网络贷款、现金分期、账单分期、商品分期 | MVP 聚焦消费贷/分期 |
| contractName | string | 合同标题、协议标题 | 保留原文标题 |
| contractNo | string | 合同编号、借据编号、额度借款合同/授信服务协议编号 | 可为空 |

### 3.2 金额与期限

| 标准字段 | 类型 | 常见表述 | 抽取规则 |
|---|---|---|---|
| loanAmount | number | 借款金额、贷款金额、贷款本金、授信金额、分期本金、合同本金 | 金额统一转为元 |
| actualReceivedAmount | number | 实际到账金额、实际发放金额、放款金额、提款金额、入账金额、可用金额 | 若存在先扣费，需与 loanAmount 区分 |
| creditLimit | number | 授信额度、循环额度、可用额度 | 循环贷场景使用 |
| termMonths | number | 贷款期限、借款期限、合同期限、分期期限 | 统一转月 |
| installmentCount | number | 分期期数、还款期数、期数 | 现金分期/账单分期常见 |
| drawdownDate | date | 放款日、提款日、起息日 | 现金流第0期时间 |
| maturityDate | date | 到期日、贷款到期日、最后还款日 | 可辅助生成现金流 |

### 3.3 利率与费率

| 标准字段 | 类型 | 常见表述 | 抽取规则 |
|---|---|---|---|
| nominalRate.value | number | 年利率、贷款年利率、执行年利率、日利率、月利率、每期分期利率 | 保留原始单位，并转换辅助字段 |
| nominalRate.unit | enum | annual/month/day/period | 日利率、月利率不要直接当年化 |
| nominalRate.method | enum | simple/compound/unknown | 标注单利/复利/未说明 |
| lprBasis | object | LPR、加/减基点、BP | 如“LPR+255BP” |
| installmentRate | number | 每期分期利率、分期手续费率、分期利息率 | 信用卡分期常见 |
| penaltyRate | string | 罚息、逾期利率、上浮50%、日利率万分之五 | 原文保留，给 C 同学判断风险 |

### 3.4 还款方式

| 标准枚举 | 常见表述 | 现金流含义 |
|---|---|---|
| equal_installment | 等额本息、按月等额还款 | 每期还款额固定，包含本金和利息 |
| equal_principal | 等额本金、等本递减 | 每期本金固定，利息随剩余本金减少 |
| interest_first_principal_later | 先息后本、按月结息到期还本、分期付息到期还本 | 中间付息，到期还本 |
| bullet | 到期一次性还本付息、利随本清 | 到期一次性流出本息 |
| equal_principal_fee | 等本等费、等本等息 | 每期本金和手续费/利息按合同固定 |
| revolving_daily_interest | 随借随还、按日计息、循环额度 | 需要实际借款天数和还款日 |

### 3.5 费用项目

费用抽取必须单列，不要混入利息。

| 标准费用类型 | 常见表述 | 是否进入成本测算 |
|---|---|---|
| interest | 利息、分期利息 | 是 |
| service_fee | 服务费、平台服务费、综合服务费 | 是 |
| management_fee | 管理费、账户管理费 | 是 |
| consulting_fee | 咨询费、信息服务费 | 是 |
| guarantee_fee | 担保费、增信服务费、保证保险费 | 是，若与贷款直接相关 |
| insurance_fee | 保险费、保费 | 是，若贷款强绑定或为获贷必要成本 |
| installment_fee | 分期手续费、每期手续费 | 是 |
| prepayment_fee | 提前还款手续费、提前结清违约金 | 或有成本；提前还款情景计算 |
| overdue_penalty | 逾期罚息、逾期违约金、复利 | 或有成本；不纳入正常履约主IRR，但要输出给 C |
| cancellation_fee | 撤销手续费、取消手续费 | 或有成本 |

### 3.6 重要条款类型

| clause.type | 要抽取的内容 | 下游用途 |
|---|---|---|
| fee | 服务费、管理费、分期利息、增信费、是否先扣 | 成本测算、费用不透明风险 |
| repayment | 还款方式、还款日、还款计划 | 现金流生成 |
| prepayment | 是否允许提前还款、手续费、违约金、申请条件 | 成本测算情景、风险识别 |
| overdue | 罚息、复利、违约金、清偿顺序 | 风险识别 |
| autoDebit | 自动扣款、还款账户授权、不可撤销授权 | 风险识别 |
| purpose | 贷款用途限制，不得购房/投资/经营等 | 建议生成 |
| rateAdjustment | LPR调整、分段利率、优惠取消 | 成本测算提醒 |
| guarantee | 担保、保证保险、增信服务 | 费用和风险 |

## 4. 合同解析 Agent 输出要求

- 每个关键字段都要带 `evidenceText`，即合同原文片段。
- 能定位时提供 `location`，如页码、条款号、段落号。
- 金额统一转为人民币元，百分比统一转为数值，例如 12.8 表示 12.8%。
- 日利率/月利率/每期费率必须保留原始单位，不能直接当年利率。
- 对 OCR 模糊或冲突字段，输出 `confidence` 和 `needsReview=true`。
- 缺失字段进入 `missingFields`，禁止编造。

## 5. MVP 最小字段

B 同学首版至少稳定输出：`loanAmount`、`actualReceivedAmount`、`termMonths/installmentCount`、`nominalRate`、`repaymentMethod`、`monthlyPayment/还款计划`、`fees[]`、`prepaymentClause`、`overdueClause`、`autoDebitClause`。

## 7. 第三批PDF合同材料补充

第三批新增了中文命名的 PDF 原始材料，重点来自中国银行、农业银行、华夏银行合同公示页，以及少量江苏银行合同直链。有效新增 PDF 主要覆盖：

- 个人贷款合同主模板；
- 个人网络贷款、线上贷款、随借随还/额度用款合同；
- 个人消费贷款、汽车消费贷款、留学/助学贷款合同；
- 个人消费贷款贴息协议、授后变更协议、续贷协议；
- 抵押、质押、保证、最高额担保等担保合同；
- 华夏银行个人消费贷款借款合同；
- 信用卡/分期协议 PDF 的补充引用。

这些材料的意义是：合同解析 Agent 不只识别“借款金额/期限/利率”，还要能识别额度、支用、贴息、变更、续贷、提前还款、担保责任、质押/抵押、罚息/复利等条款。具体文件见 `第三批PDF补充目录.md` 和 `source_catalog.csv`。
