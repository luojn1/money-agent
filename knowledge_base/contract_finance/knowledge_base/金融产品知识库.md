# 金融产品知识库（B同学：成本测算 Agent）

## 1. 目标

沉淀消费金融产品的公开产品要素、利率/费用口径、还款方式和成本计算规则，用于判断合同字段是否完整、生成现金流，并为 C 同学的风险识别提供成本类事实。

## 4. 费用进入成本测算的规则

### 4.1 正常履约主口径

进入主 IRR 的项目：

- 利息、分期利息；
- 分期手续费；
- 与贷款直接相关且正常履约也要承担的服务费、管理费、咨询费；
- 为获得贷款或降低风险而强绑定的增信服务费、担保费、保证保险费；
- 放款前扣除的费用，应减少 `actualReceivedAmount`。

不进入正常履约主 IRR，但要输出给风险识别的项目：

- 逾期罚息；
- 逾期违约金；
- 催收费、诉讼费、律师费等争议/违约情形费用；
- 提前还款手续费，除非用户选择提前还款情景测算。

### 4.2 现金流生成规则

| 还款方式 | 现金流规则 |
|---|---|
| 等额本息 | 第0期为实际到账金额；每期为固定还款额；若未给月供，按本金、月利率、期数计算 |
| 等额本金 | 每期本金=本金/期数，利息=剩余本金*月利率；每期还款递减 |
| 先息后本 | 中间各期付利息，到期还本金+最后一期利息 |
| 到期一次还本付息 | 第0期到账，到期一次性流出本金+利息 |
| 等本等费/等本等息 | 每期本金固定，手续费/利息固定或按合同还款计划 |
| 随借随还按日计息 | 需要实际借款天数和还款日期；MVP 可先返回“需还款计划” |
| 信用卡分期 | 每期本金=分期本金/期数；每期分期利息=分期本金*每期费率，或按协议给定还款计划 |

## 5. 成本测算输出建议

成本测算 Agent 必须输出：`actualReceivedAmount`、`totalRepayment`、`totalInterest`、`totalFees`、`extraCost`、`irrMonthly`、`realAnnualRateSimple`、`realAnnualRateCompound`、`displayAnnualRateMethod`、`cashFlows[]`、`includedFees[]`、`excludedContingentCosts[]`、`missingFields[]`、`assumptions[]`。

## 6. MVP 先做的三种场景

1. 固定月供消费贷：合同给出本金、期数、月供。
2. 先扣费/砍头息：合同金额大于实际到账金额，或服务费放款时一次性扣除。
3. 信用卡/消费分期：分期本金、期数、每期分期利息或每期费率。
